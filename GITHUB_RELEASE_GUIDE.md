# How to publish a GitHub Release

## Step 1 — Prepare the two distribution files

### File 1: QuantAutomationSuite.exe
Already built at:
```
FWMO\dist\QuantAutomationSuite.exe
```

### File 2: FWMO_source.zip
Create a ZIP of the source folder. Include these files only:

```
FWMO_source.zip
├── control_center.py
├── milk6_13.py
├── oranges_1_0.py
├── fishing_V2_2.py
├── fishing_v1.py
├── wood.py
├── cursor.py
├── config.json
├── requirements.txt
├── README.md
└── CHANGELOG.md
```

**Do NOT include:** `build/`, `dist/`, `_fwmo_scripts/`, `*.spec`, `*.bat`, `__pycache__/`

To create the ZIP in Windows:
1. Select all the files listed above in File Explorer
2. Right-click → **Compress to ZIP file**
3. Rename it to `FWMO_source.zip`

---

## Step 2 — Push code to GitHub

If you haven't set up the repo yet:

```bash
git init
git add .
git commit -m "Release v1.2"
git branch -M main
git remote add origin https://github.com/YOUR_USERNAME/quant-automation-suite.git
git push -u origin main
```

If repo already exists:
```bash
git add .
git commit -m "Release v1.2"
git push
```

---

## Step 3 — Create the Release on GitHub

1. Go to your repository on github.com
2. Click **Releases** (right sidebar) → **Create a new release**
3. Click **Choose a tag** → type `v1.2` → click **Create new tag: v1.2**
4. Set **Release title:** `Quant Automation Suite v1.2`
5. In the description box, paste this:

---

```
## Quant Automation Suite v1.2

Game automation toolkit for RageMP / GTA V RP.

### ⬇️ Downloads

| File | Description |
|------|-------------|
| `QuantAutomationSuite.exe` | **Recommended.** Single EXE, no Python needed |
| `FWMO_source.zip` | Python source for developers |

### 🆕 What's new in v1.2
- Built-in Cursor Logger with floating tooltip — calibrate coordinates without editing files
- Single EXE distribution — no Python required on target machine
- Fixed script path resolution (no more "script not found" errors)
- Improved launcher stability

### ⚠️ Note
Windows Defender may flag the EXE due to keyboard/mouse simulation.
Click **More info → Run anyway** or add an exclusion.

See [CHANGELOG.md](CHANGELOG.md) for full history.
```

---

6. Scroll down to **Attach binaries**
7. Drag and drop both files:
   - `QuantAutomationSuite.exe`
   - `FWMO_source.zip`
8. Click **Publish release**

---

## Result

Your release page will show:

```
Quant Automation Suite v1.2
├── QuantAutomationSuite.exe    ← users download this
├── FWMO_source.zip             ← devs download this
├── Source code (zip)           ← auto-generated by GitHub
└── Source code (tar.gz)        ← auto-generated by GitHub
```
